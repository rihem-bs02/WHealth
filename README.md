# 🛡️ WHealth 
Windows Defender

WHealth (Windows Health ) is a modern, premium desktop security audit and analysis application built with **PySide6**. 

By combining traditional signature-based detection with local Machine Learning (LightGBM) and local Large Language Models (Ollama), WHealth offers a comprehensive security workspace entirely on the desktop.

---

## ✨ Core Features

### 🔍 1. Scan Center (Multi-Engine File Audit)
The Scan Center provides a robust, multi-layered file scan workflow:
*   **Machine Learning Classification:** Integrates the **MalVisor LightGBM** model to evaluate PE headers and calculate malware probability.
*   **Signature Detection (YARA):** Scans files using rule-based YARA signatures.
*   **Hash Analysis:** Checks files against known threat databases.
*   **Static PE Inspector:** Automatically analyzes PE imports, section headers, and entropy to flag anomalies.
*   **Script Safety Checks:** Scans script files (`.ps1`, `.bat`, `.vbs`, etc.) for obfuscation or malicious commands.

### 🌐 2. Advanced Network Audit
Provides real-time local network endpoint monitoring and traffic pattern evaluation:
*   **Active Connection Mapping:** Identifies active sockets, processes, and remote IP locations using `psutil`.
*   **Traffic Flow Capture:** Captures and extracts CICIDS2017-style flow features locally.
*   **CICIDS ML Traffic Review:** Optionally processes captured network flows through an Isolation Forest and LightGBM model.
*   **Local AI Explanations:** Integrates with local **Ollama** models (e.g., `llama3.2`) to explain connection telemetry in plain language.

### 💻 3. System Hardening & Monitoring Tabs
*   **USB Scan:** Audits plugged-in storage devices for threats.
*   **Startup Items:** Scans active startup entries in registry hives and folder locations.
*   **Scheduled Tasks:** Audits Windows Task Scheduler tasks for persistence vectors.
*   **Process Behavior:** Monitors active processes for suspicious API/file activities.
*   **Memory Risk Audit:** Safely inspects active processes to score memory and behavior-related risks.

### 🗃️ 4. Quarantine & Reporting
*   **Secure Quarantine Manager:** Quarantines high-risk files by encrypting/renaming them in an isolated folder. Includes restore and delete operations.
*   **Plain-Language AI Reports:** Automatically generates detailed, interactive HTML and PDF reports with summaries generated by local AI.

---

## 🛠️ Technology Stack

*   **GUI Framework:** PySide6 (Qt for Python) with a premium dark/red security-themed UI, custom layout tokens, custom SVG icons, and dynamic animations.
*   **Machine Learning:** LightGBM, Scikit-Learn (Isolation Forest & pipelines).
*   **Local LLM Integration:** Ollama (default: `llama3.2:3b` with fallback to `llama3.2:1b`).
*   **System Telemetry:** `psutil`, `yara-python`.

## 🚀 Getting Started

### Prerequisites
*   Python 3.10 or higher (64-bit recommended).
*   [Ollama](https://ollama.com) installed and running (for AI security reporting).

### Installation & Run

1.  **Clone or navigate to the repository:**
    ```powershell
    cd WHealth
    ```

2.  **Create and activate a virtual environment:**
    ```powershell
    python -m venv .venv
    .\.venv\Scripts\Activate.ps1
    ```

3.  **Install dependencies:**
    ```powershell
    python -m pip install --upgrade pip
    python -m pip install -r requirements.txt
    ```

4.  **Run the application:**
    ```powershell
    python desktop_app.py
    ```

---

## 🤖 Configuring Local AI (Ollama)

To enable the automatic generation of plain-language security reports:
1.  Download and install [Ollama](https://ollama.com/).
2.  Start the Ollama server (or let WHealth launch it automatically).
3.  The application will automatically attempt to pull the default model (`llama3.2:3b`) on the first report request. Alternatively, you can pull it manually in your terminal:
    ```bash
    ollama pull llama3.2:3b
    ```

## ⚠️ Disclaimer & Security Note
*   **No File Execution:** WHealth **never** executes files during scanning; all analyses are static, rule-based, or model-based.
